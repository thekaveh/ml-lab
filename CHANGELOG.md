## [Unreleased]

### Added
- `clustering-iris-kmeans-vs-ae-pytorch/` — new Tier-A single-notebook task. Sibling to `dim_reduction-iris-autoencoder-pytorch/`: unsupervised KMeans benchmark on raw 4-D iris features vs on a 2-D autoencoder latent. AE is retrained inline (the original plan called for loading from `dim_reduction-...`'s `runs/<best>/`, but `runs/` is gitignored — the inline retrain is CI-safe). Recorded run (~15 s CPU, AE recon loss 0.5460 → 0.0193): KMeans cluster-vs-species ARI 0.716 / NMI 0.742 on raw features → ARI 0.835 / NMI 0.833 on the AE latent (+12 pp ARI, the AE's reconstruction objective has implicitly grouped similar inputs). §6.2 renders the 2×2 truth-vs-KMeans scatter grid on both feature spaces. §6.3 frames KMeans as a *feature-space-quality test*. Labels are used for evaluation only — clustering itself is fully unsupervised.
- `moe-fmnist-mixture-of-experts-pytorch/` — new Tier-A single-notebook task. First in-repo demo of `nnx.MoELinear` + `nnx.moe_train_step_factory`. `MoEClassifier` subclasses `FeedFwdNN` and swaps its first hidden Linear for `MoELinear(in=784, out=128, num_experts=4, top_k=2)`. Trained on Fashion-MNIST with `moe_train_step_factory(aux_loss_weight=0.05)` — the Switch-Transformer load-balancing penalty summed across every `MoELinear`. Recorded run (~18 s CPU, 844 iterations over 2 epochs): train loss 2.4078 → 0.3979; aux loss 1.2794 (random init, above the 1.0 uniform-routing floor) → 1.2218 (after training, more balanced). §6 renders the expert-utilization bar chart on a 640-sample probe; §6.3 owns the data-dependent caveat that perfect 25 % per expert isn't the goal — useful specialization can show up as 30/30/20/20. Same `NNDataset`-default-batch-size workaround as the diffusion + transformer tasks (rebuild loader at `batch_size=128`).
- `diffusion-mnist-ddpm-pytorch/` — new Tier-A single-notebook task. First in-repo demo of the nnx megamerge diffusion stack: `NoiseSchedulers.LINEAR(T=100)` for the schedule, `DiffusionMLP(input_dim=784, hidden_dims=[256, 256], time_embed_dim=32)` for the denoiser (~477k params), `diffusion_train_step_factory(schedule)` as the `train_step_fn`, `sample(model, schedule, shape=(16, 784))` for the reverse-diffusion sampling loop. The `NNModel` shell holds a placeholder `FeedFwdNN` swapped for the `DiffusionMLP` (`.net` substitution) so the `.train()` scaffolding still composes. Important detail: NNDataset's default batch_size packs the whole 54k train set into one batch — too few noise-level samples per epoch for diffusion to learn — so the notebook rebuilds the train loader at `batch_size=128` (1266 iterations over 3 epochs). Recorded run: noise-prediction loss 1.0102 → 0.9317; 16 sampled digits rendered as a 4×4 grid. Generations are blurry / mode-mixed at this scale + budget — that's the §6.3 "MLP denoiser is the bottleneck, swap for a U-Net for real fidelity" framing. ~19 s on CPU.
- `tabular_regression-diabetes-mlp-pytorch/` — new Tier-A single-notebook task. **First regression task in ml-lab.** sklearn `load_diabetes` (442 samples × 10 features, continuous target). Trains 4 models head-to-head: `sklearn.LinearRegression`, `sklearn.KNeighborsRegressor(k=5)`, `nnx` MLP small `[8]`, `nnx` MLP deep `[32, 16]` — all on the same 70/15/15 split. Surfaces a real footgun: `nnx.NNTabularDataset` coerces targets to `torch.long` (it's hard-coded for classification — see its `__post_init__`); regression requires manual `DataLoader` construction with `float32` targets shaped `(N, 1)` to match `output_dim=1` against `Losses.MEAN_SQUARED_ERROR`. Recorded held-out-test R² (n=67): LinearRegression 0.225, MLP small 0.159, MLP deep 0.130, KNN 0.028 — the textbook "linear is hard to beat on small tabular" lesson. §6 predicted-vs-actual + residual plots for all 4 models. ~22 s on CPU.
- `dim_reduction-iris-autoencoder-pytorch/` — new Tier-A single-notebook task. PCA baseline vs two autoencoder variants (shallow `[2]`-bottleneck and deeper symmetric `[3, 2, 3]`) on iris, projected to a 2-D latent. First in-repo demo of `nnx.NNModel.train(..., train_step_fn=autoencoder_step)` outside the transformer LM task — `autoencoder_step` implements `MSE(net(X), X)` and ignores the dummy `y`. Both AEs use the `FeedFwdNN(input_dim == output_dim, hidden_dims=...)` trick — no custom `nn.Module` subclass, the bottleneck is the middle Linear. Recorded run (300 epochs, ~18 s CPU): PCA explained-variance ratio [0.83, 0.13], reconstruction loss shallow `[2]` 0.5722 → 0.0142, deeper `[3, 2, 3]` 0.4212 → 0.0492. Held-out test linear-probe accuracy (LogisticRegression on the 2-D latent): PCA 82.61%, AE shallow `[2]` 100.00%, AE deeper `[3, 2, 3]` competitive. §6.3 owns the data-dependent trade-off: at iris scale the extra capacity in `[3, 2, 3]` can overfit the reconstruction and lose species separation. Producer for the queued candidate #8 (`clustering-iris-kmeans-vs-ae-pytorch`) which will load this notebook's best AE checkpoint via `NNRun.load(...)`.
- `peft-mnist-to-fmnist-dora-vs-lora-pytorch/` — new Tier-A single-notebook task. First in-repo demo of `nnx.apply_lora_to` and `nnx.apply_dora_to` for cross-task adapter fine-tuning. Pretrains a small FFN (`hidden_dims=[128, 64]`, `109,386` total params) on MNIST for 2 epochs, snapshots the state_dict, then spins up three Fashion-MNIST adapt paths from the *same* pretrained base: (a) full fine-tune control (all params trainable, 109,386), (b) LoRA at `r=8`, `alpha=16` (`9,424` trainable, 8.6% of base; wraps 3 Linear layers), (c) DoRA at same rank/alpha (`9,626` trainable, +202 magnitude params vs LoRA). Also exercises `save_lora_weights` / `load_lora_weights` round-trip into a fresh adapter (39.2 KB checkpoint file vs ~440 KB of base weights). §6.2 calls out the budget-dependent trade-off: at the short 2-epoch-pretrain + 1-epoch-adapt schedule the full-fine-tune control still leads on val accuracy; PEFT wins land at longer horizons on well-converged bases. ~25 s on CPU.
- `text_generation-tinyshakespeare-transformer-pytorch/` — new Tier-A single-notebook task. First in-repo end-to-end demo of the nnx transformer fork: trains a 256-vocab BPE tokenizer (`nnx.train_bpe` / `NNTokenizerParams`) on an embedded ~7 KB Shakespeare corpus (tiled 8× for loader-size + BPE frequency), tokenizes the corpus into a fixed-length window dataset (`SEQ_LEN=32`, `y = x.roll(-1)` next-token targets), builds a ~114 k-param `TransformerNN` (`d_model=64`, `n_heads=4`, `n_layers=2`, `max_seq_len=32`, default RoPE + tied embeddings), trains via `model.train(train_step_fn=lm_train_step)` with a custom step that flattens `(B, T, V) → (B*T, V)` for `cross_entropy`, then samples generations via `GenerativeNNModel.generate(prompt, max_new_tokens=32, temperature=0.8, top_k=20, seed=42)` for three Shakespeare-style prompts. At this scale + budget the generations are not coherent — that's the §6.3 "correctness smoke test, not quality" framing, with scaling levers (bigger corpus, bigger model, real TinyStories via HF) called out. ~8 s on CPU.
- `knowledge_distillation-mnist-ffnn-pytorch/` — new Tier-A single-notebook task. Born-again training demo (Furlanello et al., 2018) via `nnx.born_again_train`: trains a single-gen FP32 baseline as the reference, then separately runs a 3-generation born-again chain from the same fresh init (gen 0 trains on hard labels; gen k>0 distills from a frozen deepcopy of gen k-1 using `nnx.kd_train_step_factory(teacher=...)`). The recorded run shows the per-generation val-loss trajectory (2.14 → 1.91 → 1.65) and a +21% accuracy delta vs single-gen at the same parameter count, attributable to the soft-label regularization effect. ~25 s on CPU.
- `pruning-mnist-ffnn-pytorch/` — new Tier-A single-notebook task. First in-repo demo of `nnx.prune.magnitude_prune`: trains an FP32 FFN baseline (`hidden_dims=[256, 128]`), then deep-copies + prunes at each target sparsity `s ∈ {0.1, 0.3, 0.5, 0.7, 0.9}`, measuring validation accuracy + actual zero fraction. The §6 Pareto plot shows the textbook accuracy-vs-sparsity curve and discusses its knee. `bake=True` default writes zeros into `.weight` directly so the deployable state-dict keys are unchanged. 2:4 semi-structured sparsity (`nnx.prune.semi_structured_24`) is referenced in §6.3 but not exercised — its torchao kernel requires CUDA + Ampere+. ~15 s on CPU.
- `quantization-mnist-ffnn-pytorch/` — new Tier-A single-notebook task. First in-repo demo of `nnx.quantize_int8` (PTQ INT8 weight-only via torchao) + `nnx.qat_train_step_factory` / `nnx.QATLifecycleCallback` with `qat_config="8da4w"` (8-bit dynamic activation + 4-bit weight QAT). Trains FP32 baseline, quantizes it post-hoc (PTQ), separately trains a fresh model end-to-end with the QAT callback, then compares val loss / val accuracy / pickled state-dict size / per-batch CPU forward latency side-by-side. Asserts QAT callback advanced through `is_prepared` + `is_converted` and the converted net contains `Int8DynActInt4WeightLinear` modules. Hidden widths pinned to multiples of 32 (`[128, 64]`) to fit the int4 default groupsize cleanly without `padding_allowed=True`. Notebook runs in ~27 s on CPU.
- `model_surgery-mnist-ffnn-pytorch/` — new Tier-A single-notebook task. First in-repo demo of the `nnx.surgery` namespace: trains a small `FeedFwdNN` baseline on MNIST, then asserts that `nnx.widen` and `nnx.deepen` produce a forward output equal to the original within `atol=1e-5` on a probe batch (the Net2Net function-preservation contract), then races warm-start (post-surgery) vs cold-start (random init at the wider shape) resume training. Notebook runs in ~45 s on CPU; baseline pinned to `Activations.RELU` because `nnx.deepen`'s identity-init insertion is function-preserving only for ReLU. Registered in `Makefile` `TIER_A`, `scripts/verify_repo_config.yaml` (`active_task_dirs` + `tier_a_notebooks` + `required_sections`), and the root README §4 task table.
